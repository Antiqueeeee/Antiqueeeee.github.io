利用 Github Actions 自动部署 Hexo 博客

搭建流程

由于Github提供了免费的Page服务，所以就有了Hexo生成站点然后部署在Github上的玩法。博客的页面设计、部署等问题可以通过Hexo一键完成，作者只需要关注文章内容即可。当然，如果对页面美观性有一定要求可以通过其他工具实现，本篇博文只对Windows操作系统的博客搭建过程进行记录，其他需求还请读者自行探索。

丑话说在前

编写文章后，作者可以使用Hexo一键将编辑过的文章推送到Github页面上，即使没有私有服务器，其他人也可以访问到作者的博客。但Hexo并未提供“删除博客”的功能，这意味着，只要在本地项目中删除.md文件后，重新上传项目即可实现“删除”功能。换句话说，如果作者用的是新设备，且本地没有历史备份时，上传新博文等同于删除以往所有博文。
解决方案肯定是有的，本文标题中的Appveyor就是解决方案之一，其思路是，以持续集成（Continue Integration, CI）的方式完成博客内容的版本管理，就可以避免“误删”历史博文的问题。实现方法说起来也简单，首先创建一个新的repo，将新博文push到repo中，其次，Appveyor等CI工具就可以将更新的内容推送到博客站点当中。道理我都懂，但是没操作出来。Google了一圈，市面基于Hexo的博客搭建教程，大部分都只教到了访问“用户名/github.io”就能看到博客的程度。少部分在博文中提到了CI工具，但是基本上也都是从一篇2016年的博客中粘贴出来的，Appveyor的页面也变化许多，与博客内容不一致，再加上笔者本身不了解Git….
所以，本篇博文与市面上的Hexo博客搭建教程一样，只会教到访问“用户名/github.io”就能看到博客的程度，区别在于，等我研究明白CI之后我会回来填坑，他们可不一样，他们连坑都没挖。

言归正传

下面正式开始介绍如何使用Hexo + Github Page + (Appveyor)构建个人博客。首先需要安装Hexo的相关环境，是使用Hexo生成项目，然后创建Github仓库，并将Hexo生成的项目上传至Github中，最后，编写几个Markdown文件充当测试用例查看效果。

Hexo

Hexo的安装方法可以直接参考Hexo官方文档，为保证博文结构完整性，将官方文档中的相应步骤抄录至本文中。

安装前提

安装 Hexo 相当简单，只需要先安装下列应用程序即可：
Node.js (Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本)
Git
两者的安装过程中并无任何特殊配置，只需要点击“下一步”完成安装即可，读者可以根据自身喜好在安装过程中进行相应设置。安装完成后，可以在CMD中通过以下指定，测试是否安装成功。

安装Hexo

所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo，过程可能需要持续若干分钟。

安装完毕后，可以在CMD中使用hexo -v指令查看hexo版本以确定是否安装成功。

安装阶段结束后开始建站。首先需要创建一个空文件夹作为博客的项目路径。进入博客项目路径，空白处点击右键，选择“Git Bash Here”。

输入init指令，生成项目相关文件。

执行完毕后，使用hexo s指令就可以在浏览器中输入“localhost:4000”访问博客，看到博客内容。

小tips

1.post_asset_folder
当博文中需要引入图片等资源时，可以将_config.yml中的post_asset_folder参数设置为true。修改配置文件后，hexo new在创建md文件的同时还会为该博文创建一个同名的文件夹，用以存放博文中所需要的资源。要注意在博文中引入资源的方式，引入方法如图所示，使用传统的“叹号中括号括号”的引用方式是无效的。

2.创建类别、标签
网上很多hexo创建类别的教程，笔者在最开始创建分类时陷入了一个误区，就是觉得要“先生成一个类别，然后在博文中才可以设置为这个类别”，其实不是。
实际上只要按照教程操作之后，你就可以任意的为你的博文添加类别了，hexo new page categories更“像是”允许作者为博文添加类别的开关。
3.引用历史博文
详见如何在Hexo的博文中引用自己的文章，要注意中间写的是文件名，不是博文title名。
4.博文分页
当按类别访问博文且该类别博文数量较多时，Hexo会根据博客数量进行分页，默认一页内摆放十篇博文，可以在_config.yml中的per_page属性进行修改。
